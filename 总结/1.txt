**项目名称：** 天机学堂
**项目简介：** 基于 **SpringCloud Alibaba** 的在线职业技能培训平台，面向学员端与管理端，围绕课程学习、互动问答、点赞、签到积分、积分排行榜、优惠券发放/领取/使用等业务展开，采用微服务架构实现学习服务、评价服务、促销服务等模块解耦。 

**技术栈：** SpringBoot、SpringCloud、MyBatis/MyBatis-Plus、MySQL、Redis、Redisson、RabbitMQ、XXL-JOB、Caffeine、Nginx、腾讯云 VOD。

**项目内容（全量版）：**

* **学习中心 / 我的课表模块**：负责学习服务中课表能力开发，设计 **learning_lesson** 表保存用户与课程关系，核心字段包括课程学习状态、学习计划状态、周学习频率、已学习小节数、最近学习小节、最近学习时间、过期时间等；实现 **支付/报名后监听 MQ 加入课表、分页查询我的课表、查询最近正在学习的课程、按课程 id 查询学习状态、删除课表课程、退款后监听 MQ 移除课表、校验课程是否有效、统计课程学习人数** 等接口，并通过远程调用课程服务补充课程名称、封面、总小节数等展示信息。 

* **学习计划与学习进度统计模块**：基于课表和学习记录表实现学习监督能力，支持设置课程周学习频率，按周统计 **本周计划学习总量** 与 **本周已完成小节数**；实现 **查询我的学习计划** 接口，先筛选出当前用户处于计划进行中且课程状态为未开始/学习中的课表，再统计本周已完成的小节数量，最后结合课程服务返回课程信息组装 VO；为便于整体统计，采用 **查询全部后手动逻辑分页** 或基于 MyBatis 统计 SQL 的方式，分别计算总计划数和每门课的周学习进度。  

* **学习记录 / 断点续播模块**：实现视频与考试两类学习记录提交逻辑，考试小节按一次提交即视为完成；视频小节根据 **moment 与 duration** 判断是否首次学完（学习进度超过 50% 视为完成），并联动更新课表的已学习小节数、最近学习小节、最近学习时间及课程状态；当全部小节学完后，通过远程调用课程服务比较课程总小节数，自动将课表状态更新为已学完。 

* **高并发优化 / 视频进度合并写模块**：针对前端 **每 15 秒上报一次播放进度** 导致的高频写库问题，设计 **Redis 合并写 + DelayQueue 延迟检测** 方案优化断点续播；将高频更新的播放记录先写入 Redis 缓存，缓存中保存记录 id、播放进度 moment、是否学完 finished 等字段，并按 **一个 lesson 作为一个 Redis key、多个 section 作为 Hash field** 的方式减少 key 数量；用户上报进度后提交一个延迟任务，异步线程持续消费 DelayQueue，若到期后发现 Redis 中的 moment 与任务一致，说明用户已停止播放，再持久化到数据库，否则放弃旧任务，避免无效写库，大幅降低数据库写频率和并发压力。   

* **互动问答模块**：设计 **问题表 + 回答/评论表** 两张核心表，问题表记录课程、章节、提问人、匿名/隐藏状态、回答数、最近回答信息等；回答评论表通过 **自关联** 记录回答、评论、目标回复、目标用户等关系，实现一张表承载回答与评论两类数据；支持 **提问、问题详情、回答、评论、分页查询回答/评论、管理端分页管理、隐藏问答内容、删除问题并级联删除回答评论** 等功能。新增回答/评论时除写入回复表外，还会同步维护问题的回答数、回答的评论数、最新回答 id，以及学生提问未查看状态；匿名数据不回填用户信息，非匿名则通过远程调用用户服务补全昵称、头像等展示字段。   

* **问答积分异步通知模块**：在问答新增回答或评论后，基于 **RabbitMQ** 向积分系统发送异步消息，实现业务与积分解耦；例如回答问题可产生积分，问答服务不直接落积分表，而是通过 MQ 推送积分事件，由积分服务统一消费并记录积分明细。 

* **通用点赞系统模块**：将点赞能力从具体业务中抽离为独立服务，满足 **通用性、独立性、高并发、幂等防重** 要求；使用 **Redis Set** 保存点赞关系，key 为业务类型+业务 id，value 为点赞用户 id，通过 **SADD / SREM / SISMEMBER / SCARD** 实现点赞、取消点赞、判断是否点赞、统计点赞数；同时使用 **Redis ZSet** 保存发生变更的业务点赞总数，member 为业务 id，score 为当前点赞数，通过定时任务批量取出变更数据并通过 MQ 通知业务方更新业务表点赞数字段，降低与业务系统的耦合；同时提供 Feign 接口供业务侧批量查询当前用户点赞状态，批量场景下可用 **Pipeline** 提升读取效率。   

* **签到模块**：基于 **Redis Bitmap** 设计低存储成本签到方案，按 **用户 id + 年月** 生成签到 key，通过 `setBit` 记录某天是否签到，天然支持防重复签到；使用 `bitField` 读取从月初到当日的签到位图，再结合位运算统计连续签到天数，实现 **连续 7/14/28 天额外积分奖励**；签到成功后通过 RabbitMQ 向积分系统发送签到积分消息，完成签到与积分解耦。  

* **积分明细模块**：设计 **points_record** 表记录用户积分明细，支持学习、签到、问答等多种积分来源；通过 MQ 异步消费不同业务的积分消息，不同积分行为对应不同 routingKey；实现 **查询我的今日积分** 接口，按当天时间范围聚合各积分类型的获取值并返回每日上限，支持后续页面展示今日积分明细与进度。  

* **积分排行榜模块**：实现按自然月作为赛季的积分排行榜，当前赛季使用 **Redis ZSet** 实时维护榜单，member 为用户 id，score 为累计积分，每次新增积分时同步对 ZSet 执行 `incrementScore`，从而实时更新排名；支持 **查询当前赛季榜单、查询历史赛季列表、查询历史赛季榜单、查询当前用户排名和积分**。历史榜单采用 **按赛季分表** 存储，月初通过 **XXL-JOB** 创建赛季表、分页读取上月 Redis 榜单并持久化到对应的 `points_board_x` 表，再清理历史 Redis 数据；同时对积分明细表也采用按赛季拆分迁移，避免单表数据膨胀。    

* **优惠券管理模块**：实现管理端优惠券 **增删改查、审核、立即发放、定时发放、暂停发放、重新发放、结束发放、分页查询兑换码** 等功能；支持 **手动领取** 与 **兑换码指定发放** 两种发放方式，并维护优惠券状态流转：待发放、未开始、进行中、已结束、暂停；对于定时发放与定时结束场景，基于 **XXL-JOB 分片任务** 只扫描符合条件的优惠券并批量更新状态，避免全表扫描。  

* **优惠券兑换码生成与校验模块**：针对兑换码发放场景，设计支持 **20 亿+ 唯一兑换码、长度不超过 10、仅包含字母数字** 的编码方案；底层以 **32 位自增序列号** 为基础，结合 **多组按位加权求和验签、新鲜值 fresh、异或大质数混淆、Base32 转码** 生成 10 位兑换码；校验时通过 Base32 解码、解析签名与 payload、恢复序列号，并借助 **Bitmap** 校验兑换码是否已被使用，实现 **无需访问数据库的高效验签与防重复兑换**；考虑到批量生成兑换码较耗时，使用 **线程池异步生成兑换码**，减少发放接口阻塞时间。  

* **领券 / 我的优惠券 / 兑换优惠券模块**：设计 **user_coupon** 表记录用户与优惠券模板关系，字段包含用户 id、优惠券 id、有效期起止时间、使用状态、核销时间等；实现 **查询发放中的优惠券列表、手动领取优惠券、分页查询我的优惠券、兑换码兑换优惠券** 等用户端接口；在领券与兑换时校验优惠券发放时间、过期时间、库存、用户限领数量，并在成功后新增用户券记录，同时更新优惠券已发放数量与兑换码状态。  

* **领券并发安全模块**：针对领券过程中的 **库存超发** 与 **用户超领** 问题做并发控制；库存超发通过 **乐观锁** 方案解决，在更新优惠券已发放数量时增加库存条件判断；用户超领问题因涉及“查询已领数量 → 判断限领 → 新增用户券”三步操作，采用基于 **Redisson 的悲观锁** 保证同一用户领券流程串行化；同时分析并解决“**锁先释放、事务后提交**”导致的并发漏洞，通过调整边界为 **先加锁再开启事务、先提交事务再释放锁**，并借助 **AspectJ 暴露代理对象** 调用事务方法，修复事务失效与超领问题。  

* **领券高并发优化模块**：在基础并发安全之上，对领券链路做性能优化，将复杂写流程改造成 **Redis 资格校验 + MQ 异步写库**；先将优惠券必要字段缓存到 Redis，仅保留 **发放时间、库存、用户限领数量** 等资格校验所需数据，用户请求先在 Redis 中完成资格校验并即时扣减缓存库存，校验通过后发送领券消息，由监听器异步完成数据库写入、用户券创建与兑换码状态更新，减少同步请求中的数据库访问和加锁时长。  

* **通用分布式锁组件模块**：基于 **AOP + 自定义注解 + Redisson** 封装零侵入分布式锁工具，支持在业务方法上通过注解直接声明锁；注解中可配置 **锁名称、等待时间、自动释放时间、时间单位**，切面统一完成加锁、释放锁逻辑；进一步扩展支持 **锁类型切换**（可重入锁、公平锁、读锁、写锁），通过 **枚举 + 简单工厂模式** 创建不同锁对象；支持 **锁失败策略切换**，通过策略模式提供不重试/有限重试/无限重试、失败后结束或抛异常等多种策略；支持 **SpEL 动态锁名解析**，可根据方法参数生成锁粒度；支持通过 `autoUnlock` 控制是否自动释放锁，以实现简单限流能力。    

* **优惠券使用 / 最优优惠方案推荐模块**：实现用户下单时的优惠券推荐能力，先查询用户全部可用优惠券，再按订单总价做 **初筛**，筛掉不满足门槛的优惠券；之后根据优惠券作用范围与课程价格做 **细筛**，得到每张券可作用的课程集合，再通过 **排列组合** 生成多券方案和单券方案；基于 **策略模式** 抽象不同优惠规则（满减、每满减、折扣、无门槛），并使用 **CompletableFuture + CountDownLatch** 并行计算每个方案的优惠明细，最后按“**相同用券数量下优惠金额最大；相同优惠金额下用券数量最少**”筛选最优方案；同时支持查询订单使用过的优惠券规则描述，用于订单详情展示。
